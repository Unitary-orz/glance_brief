# News 格式指南

> 本文档定义各类 News 报告的通用格式规范与内容约束。
>
> **文档状态：** 现行有效
> **创建日期：** 2026-04-13
> **最近更新：** 2026-04-13

---

## 1. 报告整体结构

```
**📡 [报告名称] | [日期]**
━━━━━━━━━━━━━━━━━━━━
[区块1]
━━━━━━━━━━━━━━━━━━━━
[区块2]
━━━━━━━━━━━━━━━━━━━━
[区块3]
━━━━━━━━━━━━━━━━━━━━
[区块4]
```

### 格式要求

- **分隔符**：`━━`（全角省略号字符），前后各一行
- **日期格式**：`YYYY-MM-DD` 或脚本输出日期
- **区块标题**：无序号，直接 emoji + 名称
- **链接格式**：Markdown 超链接 `[名称](URL)`
- **简介包裹**：「」中文引号

---

## 2. 区块标题格式

### 主区块（无序号）

| 区块 | 标题格式 |
|------|---------|
| 生态趋势 | `**🌐 Agents生态趋势**` |
| OpenClaw 专项 | `**🦁 OpenClaw 专项**` |
| 开源趋势信号 | `**📈 开源趋势信号**` |
| 热点项目 | `**🔥 开源热点项目**` |

### 内部章节（序号 + emoji）

格式：`序号emoji 内容`（序号与 emoji 直接连接，无空格）

---

## 3. Agents生态趋势

### 格式

```
**🌐 Agents生态趋势**
  ① [趋势1]
  ② [趋势2]
  ③ [趋势3]
  ④ [趋势4]
  ⑤ [趋势5]
📊 活跃度：[综合描述]
```

### 数量规则

- 取 5 条趋势（不足时按实际数量输出）
- 来源：RSS 原文相关内容

---

## 4. OpenClaw 专项

### 格式

```
**🦁 OpenClaw 专项**
①📊 今日速览：[综合描述]
②🔥 社区热点：[顿号连接列表]
③🐛 Bug与稳定性：[顿号连接列表]
④✨ 功能请求：[顿号连接列表]
```

### 数量规则

- 可选章节，根据当日 RSS 实际内容调整
- 无变化则省略，不留空行
- 列表内容用顿号（、）连接，不换行

---

## 5. 开源趋势信号

### 格式

```
**📈 开源趋势信号**
  ① [引用RSS「趋势信号分析」第1条，精简]
  ② [引用RSS「趋势信号分析」第2条，精简]
  ③ [引用RSS「趋势信号分析」第3条，精简]
```

### 数量规则

- 只取 3 条
- 按 RSS 原文顺序精简呈现

### 数据来源

- 必须采用 RSS 原文「趋势信号分析」章节
- 不得自行编造或替换内容

## 6. 热点项目

### 6.1 分类合并逻辑

| 条件 | 处理方式 |
|------|---------|
| 该分类有 **≥2 个** 带「+X today」的项目 | 保留独立分类标题 |
| 该分类有 **<2 个** 带「+X today」的项目 | **合并**入一个「其他」组 |
| 该分类下**所有项目**都无 + today | 只显示「① 其他」 |

### 6.2 分类合并组包规则

今日不足 2 个 ★/日的分类打包展示，标题格式为：

```
🧠 大模型/训练 & 🔍 RAG/知识库
```

即所有被合并的分类在标题中全部列出，用 `&` 连接。

### 6.3 序号显隐规则

| 实际情况 | ① 最热 | ② 其他 |
|---------|--------|--------|
| 同时有 ★/日 和无 ★ 项目 | ✅ 显示 | ✅ 显示 |
| 只有 ★/日 项目 | ✅ 显示 | ❌ 不显示 |
| 只有无 ★ 项目 | **变为「① 其他」** | ❌ 不显示 |

### 6.4 项目格式

```
【有 ★/日】
① 最热：[项目名](URL)「简介」(+★数★/日)

【无 ★/日】
② 其他：[项目名](URL)「简介」
```

### 6.5 ★/日 标记规则

- **仅标记原文中有「+X today」的项目**
- 无 ★/日 的项目不标注数字
- 数字格式：`(+数字★/日)`，无空格

### 6.6 无 ★/日 项目的列表格式

当同一分类下有多个无 ★/日的项目时，用顿号连接：

```
② 其他：名「简介」、名「简介」、名「简介」
```

---

## 7. 序号显隐规则

### 分类内

- 同时有 ★/日 和无 ★ 项目 → 显示「① 最热」和「② 其他」
- 只有 ★/日 项目 → 只显示「① 最热」
- 只有无 ★ 项目 → 只显示「① 其他」

### OpenClaw 专项

- 章节为可选，根据 RSS 实际内容决定是否显示
- 无变化则省略，不留空行

---

## 8. 通用样式规范

| 元素 | 规范 |
|------|------|
| **标题加粗** | 主区块标题用 `**加粗**` |
| **序号缩进** | 2 个空格 + 序号 |
| **链接** | Markdown 超链接 `[名称](URL)` |
| **简介包裹** | 「」中文引号 |
| **★/日 格式** | `(+数字★/日)`，无空格 |
| **列表分隔** | 顿号（、）连接同一类目下的多个项目 |
| **分段分隔** | `━━`（前后各空一行） |

---

## 9. 缺失数据处理

| 情况 | 处理方式 |
|------|---------|
| 模板字段无数据 | 写「信息有限，待确认」 |
| 脚本执行失败 | 发「⚠️ 报告获取失败，请检查网络」 |
| 趋势不足预期数量 | 按实际数量输出，不强制凑数 |
| 某章节无内容 | 该章节整个省略，不留空行 |
| 分类内只有无 ★/日 项目 | 只显示「① 其他」 |

---

## 10. 发送前审核要点

发送前逐项核对：

- [ ] **日期**是否为正确日期
- [ ] **★/日 数字**是否与 RSS 内容中的实际数据一致
- [ ] **★/日 标记**是否仅限原文有「+X today」的项目
- [ ] **合并分类**是否正确（凡 <2 个 ★/日的分类都已合并）
- [ ] **序号显隐**是否正确
- [ ] **链接**是否为真实 URL
- [ ] **OpenClaw 专项**是否用顿号连接而非换行
- [ ] **趋势信号**是否引用 RSS 原文
